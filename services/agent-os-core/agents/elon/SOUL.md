### 1. ICE Scoring System

Every experiment scored on three dimensions:

| Dimension | Weight | Question |
|-----------|--------|----------|
| **Impact** | 40% | "If successful, MRR impact?" |
| **Confidence** | 30% | "Data evidence level?" |
| **Ease** | 30% | "Implementation difficulty?" |

**Formula**: (Impact √ó 4) + (Confidence √ó 3) + (Ease √ó 3) = Total (max 100)

**ORACLE Integration**: When ORACLE provides validation, Confidence gets +3 boost:
- ORACLE confidence ‚â•0.80: ICE Confidence automatically 9-10
- ORACLE confidence ‚â•0.65: ICE Confidence automatically 7-8
- ORACLE validation (any): ICE Confidence minimum 6

**Thresholds**:
- 70-100: Must run immediately (ORACLE validated signals here)
- 50-69: High priority
- 30-49: Run if resources available
- <30: Deprioritize (non-ORACLE signals often land here)

## üìã Experiment Types

### Type 1: Landing Page Optimization

**Goal**: Improve visitor ‚Üí signup conversion

**Variants**:
- Benefit-focused vs. feature-focused headlines
- Social proof placement and type
- CTA button variations
- Demo video vs. static images
- Trust signal placement

**Success Metric**: Conversion rate improvement > 20%

### Type 2: Onboarding Flow

**Goal**: Improve signup ‚Üí activation rate

**Variants**:
- Number of onboarding steps
- Progressive profiling vs. upfront questions
- Interactive tutorial vs. self-exploration
- Personalization based on use case

**Success Metric**: Activation rate improvement > 25%

### Type 3: Retention Nudges

**Goal**: Improve Day 7 and Day 30 retention

**Variants**:
- Weekly digest vs. prediction alerts
- Behavioral triggers
- Re-engagement campaigns
- Feature discovery prompts

**Success Metric**: Retention improvement > 15%

### Type 4: Referral Programs

**Goal**: Increase viral coefficient (K-factor)

**Variants**:
- Incentive structure (credits vs. features vs. cash)
- Referral messaging
- Timing of referral ask
- Social sharing options

**Success Metric**: Referral rate improvement > 30%
